Описание

Минимальные требования

Для работы со сборкой требуется установить Node.js, Git.

Используемые технологии

Pug, Gulp, Git, Node.js

Быстрый старт

Клонировать репозиторий с GitHub

git clone https://github.com/sergeybarbolin/mail-build

Перейти в директорию проекта

cd src

Установка зависимостей

npm install

Запуск сервера или быстрая настройка

Быстрая настройка

Отредактировать файл variables.pug в соответствии с задачей (заполнить тему и прехэдер).

mail-build/
	src/
		variables.pug

Отредактровать файл index.pug в соответствии с задачей (вы-брать тип хэдера, футера, баннеров, настроить отступы).

mail-build/
	src/
		index.pug

Добавить ЮИ в файле src/blocks/product-card.pug, вставить баннер между ЮИ если это необходимо.

mail-build/
	src/
		blocks/
			product-card.pug

Собрать верстку (из папки src)

gulp build

Вернуться в папку mail-build, там появится файл готовой рассылки index.html.

mail-build/
	src/
	index.html

Запуск сервера

Запуск сервера осуществляется из папки src/ командой

gulp watch

По умолчанию в браузере откроется вкладка с развернутым локальным сервером. Сборка происходит при каждом сохранении файлов .pug.

Необходимо повторить действия из предыдущего пункта, отслеживая изменения в реальном времени.

Сборка

Файловая структура

mail-build/ 											#Папка проекта
	src/ 												#Папка с исходниками, из которых собирается проект
		blocs/ 											#Папка с блоками
			header/ 									#Папка отдельного блока
				header.pug 								#разметка блока header
				header--new.pug 						#модифицированная разм
			product-card.pug 							#карточки товаров
		img/ 											#Папка с используемыми изображениями
			tiny/ 										#Неоптимизированные изображения
				неоптимизированные_изображение.jpg
				неоптимизированные_изображение.png
			оптимизированное_изображение.jpg
			оптимизированное_изображение.png
		node_modules 									#node модули необходимыми для сборке
		gulpfile.js 									#инициализация модулей и описание задачи
		index.pug    									#файл рассылки
		mixins.pug 										#Примеси для более быстрого и комфортного написания кода
		style.pug 										#CSS стили
		variables.pug 									#Переменные

Принцип работы

Итоговая html верстка собирается из отдельных блоков:
2 типа хэдера
2 типа навигации в шапке
2 типа баннеров «Зайди в магазин, не выходя из дома»
2 типа навигации в футэре
2 типа футера

Также написаны миксины для быстрого добавления баннеров, карточек товаров, отступов разной величины.

Необходимые блоки подключаются через include в файле index.pug.

Для добавления баннера или карточки товара необходимо вызвать миксин с необходимыми параметрами. Более подробно можно узнать в разделе Карточки товара или Баннеры.

Баннеры можно вставлять в любое место.

Карточки товаров добавляются в файле src/blocks/product-card.pug, там же можно вставить баннер между ЮИ.

Стили из файла style.pug добавляются автоматически при сборке к соот-ветствующим тегам и классам.

Все изображения в папке src/img/tiny с разрешением .png, .jpg, .jpeg авто-матически сжимаются без потери качества и переносятся в папку src/img/. В режиме gulp watch при добавлении изображений в папку src/img/tiny они также сжимаются и переносятся в src/img/ в реальном времени.

В режиме gulp watch разворачивается локальный сервер и при каждом со-хранении любого файла .pug происходит сборка и авто обновление стра-ницы в браузере, таким образом вы видите все изменения в реальном вре-мени.

Команда gulp build производит такую же сборку как gulp watch, но всего один раз, без развертывания локального сервера. (Работает быстрее)

После сборки в корневой папке mail-build появляется файл рассылки index.html.
